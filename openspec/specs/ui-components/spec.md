## 需求
### Requirement: 应用主题系统

系统 MUST 定义统一的视觉主题，符合多邻国风格设计。

#### Scenario: 配色方案定义

- **当** 应用主题加载
- **那么** 包含以下颜色定义：
  - 正确绿: #58CC02
  - 错误红: #FF4B4B
  - 主题蓝: #1CB0F6
  - 背景白: #FFFFFF
  - 浅灰背景: #F7F7F7
  - 边框灰: #E5E5E5
  - 次要文字: #777777
  - 主要文字: #3C3C3C

#### Scenario: 字体样式定义

- **当** 渲染文本
- **那么** 使用定义的字体层级（标题、正文、按钮、标签）

### Requirement: 首页

系统 MUST 提供首页，展示主题选择列表。

#### Scenario: 显示主题列表

- **当** 进入首页
- **那么** 显示可选主题列表（JavaScript、React、Vue 等）

#### Scenario: 选择主题进入答题

- **当** 点击某个主题
- **那么** 导航到答题页并加载该主题题目

### Requirement: 答题页

系统 MUST 提供答题页，展示题目和选项，支持答题交互。

#### Scenario: 显示题目进度

- **当** 进入答题页
- **那么** 显示进度条（当前题号/总题数）

#### Scenario: 显示题目内容

- **当** 加载题目完成
- **那么** 显示题目内容、难度标签、标签列表

#### Scenario: 显示选项列表

- **当** 题目加载完成
- **那么** 显示 4 个选项卡片（A/B/C/D）

#### Scenario: 选择选项

- **当** 点击某个选项
- **那么** 该选项高亮显示（蓝色边框）
- **并且** 其他选项恢复默认状态

#### Scenario: 提交答案

- **当** 已选择选项并点击提交按钮
- **那么** 显示答案反馈
- **并且** 正确选项显示绿色
- **并且** 错误选择显示红色

### Requirement: 答案反馈面板

系统 MUST 在提交答案后显示反馈面板。

#### Scenario: 正确答案反馈

- **当** 答案正确
- **那么** 显示"回答正确!"提示
- **并且** 背景为浅绿色

#### Scenario: 错误答案反馈

- **当** 答案错误
- **那么** 显示"回答错误"提示
- **并且** 背景为浅红色
- **并且** 显示正确答案

#### Scenario: 显示答案解析

- **当** 反馈面板展开
- **那么** 显示详细解析内容
- **并且** 显示助记口诀（如有）
- **并且** 显示实战场景（如有）

### Requirement: 结果页

系统 MUST 在完成一轮答题后显示结果页。

#### Scenario: 显示答题统计

- **当** 进入结果页
- **那么** 显示总题数、正确数、错误数、正确率

#### Scenario: 重新开始

- **当** 点击"重新开始"按钮
- **那么** 返回答题页并重新加载题目

#### Scenario: 返回首页

- **当** 点击"返回首页"按钮
- **那么** 导航到首页

### Requirement: 选项卡片组件

系统 MUST 提供 OptionCard 组件，支持多种状态展示。

#### Scenario: 默认状态

- **当** 选项未被选中
- **那么** 显示灰色边框和灰色选项字母

#### Scenario: 选中状态

- **当** 选项被选中但未提交
- **那么** 显示蓝色边框和蓝色选项字母

#### Scenario: 正确状态

- **当** 提交后该选项是正确答案
- **那么** 显示绿色边框、浅绿背景、绿色选项字母

#### Scenario: 错误状态

- **当** 提交后该选项是错误选择
- **那么** 显示红色边框、浅红背景、红色选项字母

### Requirement: 进度条组件

系统 MUST 提供 ProgressBar 组件，显示答题进度。

#### Scenario: 显示当前进度

- **当** 答题进行中
- **那么** 显示"第 X 题 / 共 Y 题"
- **并且** 进度条填充比例为 X/Y

### 需求：交互动效与触感反馈

系统 MUST 为关键交互提供一致的动效与触感反馈，强化“斩题”体验。

#### 场景：选项按压反馈

- **当** 用户按下 OptionCard
- **那么** OptionCard 产生短暂下沉位移并伴随阴影/边框的过渡
- **并且** 触发轻触感反馈

#### 场景：提交后反馈面板动效

- **当** 用户提交答案
- **那么** FeedbackPanel 以弹性曲线从底部进入
- **并且** 面板内容支持长文本滚动

### 需求：多尺寸适配

系统 MUST 在不同屏幕尺寸与横竖屏下保持可读性与交互可达性。

#### 场景：小屏可读性

- **当** 设备宽度较小（例如 <= 360dp）
- **那么** 题干与解析文本保持可读行高
- **并且** 关键操作按钮在可视区域内可达（无需精细滑动寻找）

#### 场景：平板/大屏布局

- **当** 设备宽度 >= 600dp
- **那么** QuestionPage 采用更宽松的布局（例如双栏或更大留白）
- **并且** 题干区域的最大阅读宽度受到限制，避免超长行影响阅读

### 需求：加载与异常状态体验

系统 MUST 提供与主题一致的加载、错误与空状态体验。

#### 场景：加载骨架

- **当** QuestionPage 处于 loading 状态
- **那么** 显示与卡片圆角一致的骨架占位

#### 场景：网络错误

- **当** 题目加载失败
- **那么** 显示友好的错误文案与“重试”入口

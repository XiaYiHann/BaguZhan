# 变更：M1 基础 UI 原型（固定数据）

## 为什么

项目当前只有文档和参考 HTML，没有实际可运行的应用。需要构建 Flutter 客户端基础框架，使用固定数据实现完整的答题循环，验证核心用户体验后再对接后端。

## 变更内容

- 初始化 Flutter 项目及核心依赖配置（含测试依赖）
- 建立分层架构：UI 层 / 状态管理层 / 数据访问层
- **TDD 开发**：先编写测试，再实现功能
- 实现数据模型：QuestionModel、OptionModel、UserAnswerModel
- 创建固定题库数据源（至少 8 道题）
- 实现状态管理（QuestionProvider）
- 开发核心页面：首页、答题页、结果页
- 开发核心组件：题目卡片、选项卡片、进度条、反馈面板
- 配置路由与页面导航
- 编写完整的单元测试、组件测试和集成测试

## 影响

- 受影响规范：
  - `flutter-app` (新增) - Flutter 应用基础架构
  - `question-flow` (新增) - 答题流程与状态管理
  - `ui-components` (新增) - UI 组件规范
  - `testing` (新增) - 测试规范与覆盖率要求
- 受影响代码：
  - 新建 `baguzhan/` Flutter 项目目录
  - 新建 `lib/` 下所有源码文件
  - 新建 `test/` 下所有测试文件

## 验收标准

- [ ] 可选择主题进入答题
- [ ] 可浏览题目并选择答案

- [ ] 完成一轮答题后显示统计结果
- [ ] 可重新开始或返回首页
- [ ] 所有单元测试和组件测试通过
